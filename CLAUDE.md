## Project Overview

PSX AI Stock Scanner -- a Python + Streamlit app that scans all Shariah-compliant stocks on the Pakistan Stock Exchange (PSX) and outputs top 10 T+1/swing trade picks with entry, target, stoploss, and risk:reward. Uses CrewAI agents with configurable LLM providers.

## Stack

- **UI:** Streamlit (multi-page: KMI Scanner + Stock Analyzer)
- **Historical Data:** yfinance (`.KA` suffix for PSX tickers) -- fetched once, stored locally
- **Live Data:** Sarmaaya API (`/api/stocks/` for OHLC, `/api/stocks/ticker` for volume)
- **OHLCV Store:** Per-stock parquet files in `cache/ohlcv/` (managed by `ohlcv_store.py`)
- **Technicals:** pandas + pandas-ta (RSI, EMA, MACD, ATR, volume)
- **Agents:** CrewAI
- **LLM:** Configurable -- Gemini, Groq, OpenAI, Ollama
- **Config:** `.env` file for API keys

## Data Architecture

- **OHLCV Store** (`cache/ohlcv/`): Per-stock parquet files. Initial 6mo backfill from yfinance, daily updates from Sarmaaya API via "Refresh Today" button. Max 1 year retention.
- **Scans read locally**: `fetch_all()` and `fetch_single_stock()` make zero network calls -- they read parquets and compute technicals.
- **"Generate History" button**: Deletes all parquets and re-fetches 6mo from yfinance. Used for initial setup or testing reset.
- **"Refresh Today" button**: Fetches today's OHLCV from Sarmaaya API and updates the store.

## Stock Universe

- `KMI_all.csv` -- all Shariah-compliant stocks from KMIALLSHR index (after exclusions), sorted by market cap
- Generated by `parse_kmi.py` from `KMIALLSHR.csv` + enrichment from `psx_market_watch.csv` (sector IDs), `sectors.txt` (sector names), `stocks_data.json` (company names)
- Static list, refresh every ~6 months when PSX recomposes the index
- This is the only allowed stock universe -- no AI involvement in stock selection

## Key Constraints

- **AI must never fetch prices or OHLCV data** -- hallucination risk. Data fetching is pure Python.
- **AI only touches analysis** -- CrewAI agents receive pre-computed technicals and output trade picks.
- **yfinance SSL fix** -- yfinance uses `curl_cffi` internally. Pass a `curl_cffi.requests.Session(verify=False)` to `yf.Ticker(symbol, session=session)` to bypass Windows SSL certificate issues.
- **yfinance only for history backfill** -- not called during scans. Daily live data comes from Sarmaaya API.

## CrewAI Note

CrewAI throws an error if `OPENAI_API_KEY` is not set, even when not using OpenAI. Set it to a dummy value in `.env`:
```
OPENAI_API_KEY=NA
```
