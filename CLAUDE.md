## Project Overview

PSX AI Stock Scanner — a Python + Streamlit app that scans the top 100 Shariah-compliant stocks on the Pakistan Stock Exchange (PSX) and outputs top 10 T+1 trade picks with entry, target, stoploss, and risk:reward. Uses CrewAI agents powered by Google Gemini for analysis.

## Stack

- **UI:** Streamlit
- **Data:** yfinance (`.KA` suffix for PSX tickers, e.g. `OGDC.KA`) with SSL workaround via `curl_cffi` session
- **Technicals:** pandas + pandas-ta (RSI, EMA, MACD, volume)
- **Agents:** CrewAI
- **LLM:** Google Gemini (`gemini-2.0-flash`) via `langchain-google-genai` — configurable via settings panel
- **Config:** `.env` file for API keys

## Stock Universe

- `KMI_top100.csv` — top 100 Shariah-compliant stocks from KMIALLSHR index, sorted by market cap
- Generated by `parse_kmi.py` from `KMIALLSHR.csv` (source: PSX website export)
- Static list, refresh every ~6 months when PSX recomposes the index
- This is the only allowed stock universe — no AI involvement in stock selection

## Key Constraints

- **AI must never fetch prices or OHLCV data** — hallucination risk. Data fetching is pure Python (yfinance).
- **AI only touches analysis** — CrewAI agents receive pre-computed technicals and output trade picks.
- **yfinance SSL fix** — yfinance uses `curl_cffi` internally. Pass a `curl_cffi.requests.Session(verify=False)` to `yf.Ticker(symbol, session=session)` to bypass Windows SSL certificate issues.
- **yfinance rate limits** — avoid hammering Yahoo Finance repeatedly. In production, fetch all tickers once per day after market close.

## CrewAI Note

CrewAI throws an error if `OPENAI_API_KEY` is not set, even when not using OpenAI. Set it to a dummy value in `.env`:
```
OPENAI_API_KEY=NA
```
